Module bobot

Library for using usb4butia boards.

This library interface with bobot library for accessing usb4butia-attached devices. It supports hotplug. There will be one object of this type for each bobot device. Bobot device's methods (it's api in the bobot terms) are available.

Functions

init (conf)
Initialize and starts the module. This is called automatically by toribio if the load attribute for the module in the configuration file is set to true.

Parameters:

  • conf: the configuration table (see conf).

Tables

conf
Configuration Table. This table is populated by toribio from the configuration file.

Fields:

  • load: whether toribio should start this module automatically at startup.
  • comms: communitaction parameter to provide to bobot library (such as {"usb", "serial"})
  • path: where the bobot library is installed.
  • timeout_refresh: Time before triggering a re-detection of attached modules (nil or negative disables).

Fields

name
Name of the device. Starts with 'bb-' and then the name provided by bobot. For example, "bb-dist:1".
module
Module of the device. Starts with 'bb-' and then the module provided by bobot. For example, "bb-dist".
